Analysis Method to Evaluate A Case Study September 2005 Technical Note Paul Clements John Bergey Dave Mason This report describes the application of the SEI ATAM Architecture Tradeoff Analysis Method to the Army Warfighter Information system Publisher Software Engineering Institute Report Number Subjects Software ArchitectureSoftware Architecture Abstract The software architecture of a system greatly determines system quality Evaluating that architecture for fitness of purpose before the system is implemented or undergoes a major modification is a approach to uncovering deficiencies early and reducing risk When used appropriately software architecture evaluations can have a favorable effect on a delivered or modified government system This technical note describes the application of the SEI Architecture Tradeoff Analysis Method ATAM to the Army Warfighter Information system The system is being developed by a team headquartered at the Army Communications and Electronics Command CECOM in Monmouth New Jersey This technical note presents the program context the definition of software architecture and the background of the organization and system being evaluated It also provides a general overview of the ATAM process describes the application of the ATAM to the system presents important results and summarizes the benefits the program received Share Download PDF Part of a
