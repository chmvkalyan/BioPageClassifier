that statically recovers detailed abstractions from executables in a scalable manner Publisher ACM Subjects Reverse Engineering for Malware AnalysisReverse Engineering for Malware Analysis Abstract source code abstractions such as classes and methods greatly assist human analysts and automated algorithms alike when analyzing programs Unfortunately these abstractions are lost when compiling source code which impedes the understanding of executables In this article we propose a system OOAnalyzer that uses an innovative new design to statically recover detailed abstractions from executables in a scalable manner OOAnalyzer design is motivated by the observation that many human analysts reason about programs by recognizing simple patterns in binary code and then combining these findings using logical inference domain knowledge and intuition We codify this approach by combining a lightweight symbolic analysis with a flexible reasoning system Unlike most existing work OOAnalyzer is able to recover both polymorphic and classes We show in our evaluation that OOAnalyzer assigns over 78 of methods to the correct class on our test corpus which includes both malware and software such as Firefox and MySQL These recovered abstractions can help analysts understand the behavior of malware and cleanware and can also improve the precision of program analyses on executables
